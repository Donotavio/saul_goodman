# Internacionalização (i18n)

## Extensão Chrome

- Strings em `_locales/<locale>/messages.json` (formato Chrome).
- `resolveLocale()` decide idioma com `localePreference` ou `chrome.i18n.getUILanguage()`.
- Fallback padrão: `en-US`.
- RTL automático para `ar` e `ur`.

## Site e blog

- Reutilizam `_locales/` via cópia para `site/_locales/` e `site/blog/_locales/`.
- O frontend usa `site/shared/i18n.js` para aplicar traduções.

## Extensão VS Code

- Usa `vscode-extension/_locales/<locale>/messages.json`.
- Idiomas suportados: `pt-BR`, `en-US`, `es-419` (config `saulGoodman.language`).

## Scripts úteis

```bash
npm run i18n:sync
npm run i18n:check
npm run i18n:stubs
npm run i18n:copy-site
npm run i18n:repair-locale -- --locale zh_CN
```

## Boas práticas

- Use chaves únicas e descritivas.
- Nunca coloque conteúdo dinâmico diretamente em `data-i18n-html`.
- Atualize `site/_locales` sempre que mudar `_locales`.
